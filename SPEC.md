# K-NOMAD HUB 웹사이트 개선 계획

## 프로젝트 개요

현재 웹사이트를 개선하기 위해서 실행 계획을 제작합니다. 여러 단계에 걸쳐서 계획을 구현해주고 각 단계는 Phase라고 부릅니다. Phase 1, Phase 2 등. 각 단계는 그 자체적으로 실행이 가능한 형태로 완성되어야 하고 점진적으로 프로젝트를 개선해나가게 됩니다.

각 단계별로 독립적인 에이전트가 작업 할 수 있도록 충분한 문맥을 작성해줍니다. 최소 아래 내용들을 담아줍니다.

**참고사항:**
- 아직 데이터베이스는 사용하지 않을 것이고 가짜 데이터로 진행합니다.
- 기능을 구현하지는 말고 단계별 계획을 출력합니다.

## Phase 1: 코어 구조 정리 및 내비게이션 단순화

### 이번 단계에서 개선할 사항 오버뷰

홈페이지와 인증 페이지만 남기고 불필요한 페이지들을 제거하여 웹사이트 구조를 단순화합니다. 내비게이션을 정리하고 사용자가 핵심 기능에 집중할 수 있도록 합니다.

### 수정/개선해야 하는 기능 체크리스트

- [ ] `/cities`, `/compare`, `/community`, `/guides` 페이지 삭제
- [ ] `components/layout/navigation.tsx` (또는 `Header.tsx`)에서 도시탐색, 비교하기, 커뮤니티, 가이드 내비게이션 메뉴 제거
- [ ] 인증 및 프로필 관련 버튼(로그인, 회원가입, 사용자 메뉴)만 내비게이션에 유지
- [ ] 삭제된 페이지로 향하는 모든 링크 제거 또는 홈페이지로 리다이렉션
- [ ] 사용하지 않는 컴포넌트 및 파일 정리
- [ ] 별점 평점 시스템 완전 제거 (관련 컴포넌트 및 상태 관리 코드 삭제)
- [ ] 도시 카드의 "자세히 보기" 버튼 제거

### 작업 완료 후 검증/확인 해야 하는 기능 체크리스트

- [ ] 홈페이지(`/`)가 정상적으로 로드되는지 확인
- [ ] 로그인 페이지(`/login`)가 정상적으로 작동하는지 확인
- [ ] 회원가입 페이지(`/register`)가 정상적으로 작동하는지 확인
- [ ] 내비게이션 메뉴에 인증 관련 버튼만 표시되는지 확인
- [ ] 삭제된 페이지 접근 시 적절한 처리(404 또는 홈 리다이렉션)가 되는지 확인
- [ ] 별점 시스템이 완전히 제거되었는지 확인
- [ ] 자세히 보기 버튼이 모든 카드에서 제거되었는지 확인

## Phase 3: 필터 시스템 구현 및 카드 구조 개선

### 이번 단계에서 개선할 사항 오버뷰

4가지 필터 카테고리(예산, 지역, 환경, 최고 계절)를 정의하고 UI를 구현합니다. 도시 카드 구조를 변경하여 필터 정보를 Key-Value 형태로 표시합니다.

### 수정/개선해야 하는 기능 체크리스트

- [ ] 필터 데이터 타입 정의 (TypeScript interface/type)
- [ ] 예산 필터 옵션 구현: "100만원", "100~200만원", "200만원"
- [ ] 지역 필터 옵션 구현: "전체", "수도권", "경상도", "전라도", "강원도", "제주도", "충청도"
- [ ] 환경 필터 옵션 구현: "자연친화", "도심선호", "카페작업", "코워킹 필수"
- [ ] 최고 계절 필터 옵션 구현: "봄", "여름", "가을", "겨울"
- [ ] 필터 UI 컴포넌트 구현 (체크박스, 드롭다운 등)
- [ ] 필터 섹션 레이아웃 구현
- [ ] 도시 카드에 필터 정보 Key-Value 형태로 표시하는 UI 구현
- [ ] 카드 구조 변경: 예산, 지역, 환경, 최고 계절을 Key-Value로 표시
- [ ] 카드 레이아웃 재구성 (필터 정보 공간 확보)

### 작업 완료 후 검증/확인 해야 하는 기능 체크리스트

- [ ] 4가지 필터 카테고리가 모두 표시되는지 확인
- [ ] 예산 필터의 모든 옵션이 정확히 표시되는지 확인
- [ ] 지역 필터의 모든 옵션이 정확히 표시되는지 확인
- [ ] 환경 필터의 모든 옵션이 정확히 표시되는지 확인
- [ ] 최고 계절 필터의 모든 옵션이 정확히 표시되는지 확인
- [ ] 필터 UI가 사용자 친화적으로 배치되었는지 확인
- [ ] 도시 카드에 "예산" 정보가 Key-Value로 표시되는지 확인
- [ ] 도시 카드에 "지역" 정보가 Key-Value로 표시되는지 확인
- [ ] 도시 카드에 "환경" 정보가 Key-Value로 표시되는지 확인
- [ ] 도시 카드에 "최고 계절" 정보가 Key-Value로 표시되는지 확인
- [ ] Key-Value 형태가 읽기 쉽게 표시되는지 확인
- [ ] 카드 레이아웃이 깔끔하고 정보가 잘 구조화되어 있는지 확인

## Phase 4: 가짜 데이터 수정 및 정렬 기능 구현

### 이번 단계에서 개선할 사항 오버뷰

각 도시별로 필터에 해당하는 속성을 추가한 가짜 데이터를 생성합니다. 좋아요 수를 기준으로 도시 리스트를 정렬하여 표시합니다.

### 수정/개선해야 하는 기능 체크리스트

- [ ] 기존 가짜 데이터 구조에 필터 속성 추가 (budget, region, environment, bestSeason)
- [ ] 각 도시마다 예산 정보 할당 (100만원 / 100~200만원 / 200만원 중 선택)
- [ ] 각 도시마다 지역 정보 할당 (수도권, 경상도, 전라도, 강원도, 제주도, 충청도 중 선택)
- [ ] 각 도시마다 환경 정보 할당 (최소 1개 이상: 자연친화, 도심선호, 카페작업, 코워킹 필수)
- [ ] 각 도시마다 최고 계절 정보 할당 (봄, 여름, 가을, 겨울 중 선택)
- [ ] 각 도시에 좋아요/싫어요 초기값 설정 (다양한 숫자로 설정)
- [ ] 가짜 데이터 수정: 각 도시별로 필터에 해당되는 요소 최소 하나씩 추가
- [ ] 좋아요 수 기준 내림차순 정렬 로직 구현
- [ ] 정렬된 도시 리스트를 화면에 렌더링하는 로직 구현

### 작업 완료 후 검증/확인 해야 하는 기능 체크리스트

- [ ] 모든 도시에 예산 정보가 할당되었는지 확인
- [ ] 모든 도시에 지역 정보가 할당되었는지 확인
- [ ] 모든 도시에 환경 정보가 최소 1개 이상 할당되었는지 확인
- [ ] 모든 도시에 최고 계절 정보가 할당되었는지 확인
- [ ] 도시 리스트가 좋아요 수 기준으로 내림차순 정렬되는지 확인
- [ ] 좋아요 버튼 클릭 시 정렬 순서가 동적으로 변경되는지 확인
- [ ] 데이터 구조가 타입 안전성을 유지하는지 확인 (TypeScript)
- [ ] 각 도시의 필터 정보가 카드에 정확히 표시되는지 확인

## Phase 5: 필터링 기능 구현 및 최종 통합 테스트

### 이번 단계에서 개선할 사항 오버뷰

필터 선택에 따라 도시 리스트를 동적으로 필터링하는 기능을 구현합니다. 카드 구조 변경 시 위에서 정의한 필터(예산, 지역, 환경, 최고 계절)를 Key-Value로 보여줍니다. 모든 기능이 통합되어 정상 작동하는지 최종 검증합니다.

### 수정/개선해야 하는 기능 체크리스트

- [x] 필터 상태 관리 로직 구현 (useState 또는 Context API)
- [x] 예산 필터 선택 시 해당 예산 범위에 맞는 도시만 표시
- [x] 지역 필터 선택 시 해당 지역의 도시만 표시 ("전체" 선택 시 모든 도시 표시)
- [x] 환경 필터 선택 시 해당 환경을 포함한 도시만 표시
- [x] 최고 계절 필터 선택 시 해당 계절이 최고인 도시만 표시
- [x] 다중 필터 조합 시 AND 조건으로 필터링 (모든 조건을 만족하는 도시만 표시)
- [x] 필터 초기화 버튼 구현
- [x] 필터링 결과가 없을 때 적절한 메시지 표시
- [x] 필터링 + 정렬이 함께 작동하도록 통합 (필터링 후 좋아요 순 정렬)

### 작업 완료 후 검증/확인 해야 하는 기능 체크리스트

- [ ] 예산 필터가 정상 작동하는지 확인
- [ ] 지역 필터가 정상 작동하는지 확인 (전체 옵션 포함)
- [ ] 환경 필터가 정상 작동하는지 확인
- [ ] 최고 계절 필터가 정상 작동하는지 확인
- [ ] 여러 필터를 동시에 선택했을 때 올바르게 필터링되는지 확인
- [ ] 필터링된 결과가 좋아요 순으로 정렬되는지 확인
- [ ] 필터 초기화 시 모든 도시가 다시 표시되는지 확인
- [ ] 필터링 결과가 없을 때 적절한 메시지가 표시되는지 확인
- [ ] 좋아요/싫어요 기능이 필터링 상태에서도 정상 작동하는지 확인
- [ ] 좋아요/싫어요 클릭 후 필터링 상태가 유지되는지 확인
- [ ] 전체 페이지 반응형 디자인이 정상 작동하는지 확인
- [ ] 인증 페이지(로그인/회원가입)가 정상 작동하는지 확인
- [ ] 모든 페이지에서 내비게이션이 일관되게 표시되는지 확인
- [ ] 성능 이슈 없이 필터링과 정렬이 빠르게 작동하는지 확인

## Phase 6: 검색 및 정렬 옵션 확장

### 이번 단계에서 개선할 사항 오버뷰

필터링 기능에 검색과 다양한 정렬 옵션을 추가하여 사용자가 더 쉽게 원하는 도시를 찾을 수 있도록 UX를 개선합니다. 검색바를 추가하고, 정렬 드롭다운을 구현하며, 검색 + 필터 + 정렬이 통합적으로 작동하도록 합니다.

### 수정/개선해야 하는 기능 체크리스트

- [x] 검색바 UI 컴포넌트 구현 (SearchBar)
- [x] 검색 아이콘 추가 (lucide-react)
- [x] 검색어 입력 시 실시간 필터링
- [x] 검색 결과 개수 표시
- [x] 정렬 옵션 드롭다운 구현 (SortDropdown)
- [x] 4가지 정렬 옵션 제공 (좋아요↓, 좋아요↑, 이름↑, 이름↓)
- [x] 한글 이름 정렬 지원 (localeCompare)
- [x] PopularCities에서 검색 상태 관리
- [x] PopularCities에서 정렬 상태 관리
- [x] 검색 + 필터 + 정렬 통합 로직 구현 (useMemo)
- [x] 전체 초기화 버튼 기능 확장 (검색 + 필터 + 정렬)
- [x] 빈 결과 메시지 개선 (검색과 필터 구분)

- [ ] 정렬 드롭다운이 정상적으로 표시되는지 확인
- [ ] "좋아요 많은 순" 정렬이 정상 작동하는지 확인
- [ ] "좋아요 적은 순" 정렬이 정상 작동하는지 확인
- [ ] "이름 오름차순" 정렬이 정상 작동하는지 확인 (가나다순)
- [ ] "이름 내림차순" 정렬이 정상 작동하는지 확인

## 개발 시 주의사항

1. **데이터베이스 미사용**: 모든 Phase에서 데이터베이스는 사용하지 않고 가짜 데이터(mock data)로 진행합니다.
2. **단계별 독립성**: 각 Phase는 독립적으로 실행 가능하도록 구성되어 있으며, 순차적으로 진행하는 것을 권장합니다.
3. **검증 필수**: 각 Phase 완료 후 반드시 검증 체크리스트를 확인하여 기능이 정상 작동하는지 테스트해야 합니다.
4. **타입 안전성**: TypeScript를 사용하여 타입 안전성을 유지합니다.
5. **컴포넌트 재사용**: 공통 UI 컴포넌트는 재사용 가능하도록 설계합니다.
6. **사용자 경험**: 모든 기능은 직관적이고 사용하기 쉽게 구현합니다.
7. **반응형 디자인**: 모바일, 태블릿, 데스크톱 모든 화면 크기에서 정상 작동하도록 합니다.
